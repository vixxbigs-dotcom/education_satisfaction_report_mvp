# 2차 개발 전 결정할 사항

## 1. 표준 PPT 디자인 일치 수준

현재는 목표 PPT의 정보 구조와 색상 체계를 반영한 MVP입니다. 다음 단계에서는 아래 중 하나를 정해야 합니다.

- **A안: 제공 PPT를 직접 수정·복제** — 디자인 일치도가 높지만 템플릿 구조 변화에 민감
- **B안: 코드로 동일하게 재현** — 동적 장표 생성이 안정적이지만 좌표·폰트 검수가 필요

권장안은 B안으로 레이아웃을 코드화하되, 제공 PPT를 좌표 측정 기준으로 사용하는 방식입니다.

## 2. 커리큘럼 장표

- 한 장 최대 행 수
- 행이 넘을 때 다음 장 자동 생성 여부
- Day 병합, 모듈 번호, 강사명 표기 규칙

## 3. 주관식 장표

- 원문 전체 수록 여부
- 한 장 최대 답변 수 또는 최대 글자 수
- ‘없음’류 의견의 기본 제외 여부
- 향후 AI 요약·주제 분류를 추가할지 여부

## 4. 객관식 결과

- 평균 소수점 자릿수: 1자리 또는 2자리
- 긍정 응답률(4~5점) 표시 여부
- 강사별·과정별 묶음 요약 장표 필요 여부

## 5. 사진 장표

- 빈 사진 장표를 항상 포함할지 여부
- 사진 1~6장별 레이아웃 확정
- 사진 설명 문구 입력 기능 필요 여부

## 6. 사용자 기능

- 슬라이드 포함/제외
- 장표 순서 변경
- 로고 업로드
- 포인트 컬러 선택
- 분석 결과 JSON 저장 및 재불러오기
